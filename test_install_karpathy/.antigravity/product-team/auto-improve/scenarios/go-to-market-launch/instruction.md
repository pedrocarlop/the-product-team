# Scenario: Go-to-Market Launch Plan

## Task
Create a go-to-market launch plan for a B2B analytics tool entering the SMB (small and medium business) market. The product helps SMBs understand customer behavior through no-code dashboards and automated insights. The company has an existing enterprise product but is launching a self-serve tier.

## Requirements
1. **Positioning**: Define the value proposition, target persona, and key differentiators against 2-3 named competitor categories (e.g., "spreadsheet users", "enterprise BI tools", "point analytics solutions").
2. **Channel Strategy**: Identify at least 4 acquisition channels with estimated relative priority. Include both paid and organic channels with rationale for each.
3. **Launch Timeline**: Provide a 12-week launch timeline with milestones for pre-launch, launch week, and post-launch phases. Include dependencies between milestones.
4. **Success Metrics**: Define 5+ measurable KPIs with target values for the first 90 days (e.g., "500 self-serve signups", "15% activation rate").
5. **Competitive Differentiation**: Articulate how messaging shifts for each competitor category, with specific proof points or feature advantages.

## Output Contract
Produce a `launch-plan.md` artifact with ## sections for each requirement. Include a timeline table and a metrics dashboard specification.
