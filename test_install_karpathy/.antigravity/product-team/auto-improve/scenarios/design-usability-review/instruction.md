# Scenario: Design Usability Review

## Task
Review the usability of a settings page for a project management SaaS product. The page currently presents 30+ configuration options in a single scrollable view, covering profile settings, notification preferences, workspace configuration, integrations, billing, and security. Users report difficulty finding specific settings and confusion about which changes require confirmation.

## Requirements
1. **Heuristic Evaluation**: Evaluate the page against at least 6 of Nielsen's 10 usability heuristics. For each heuristic, state whether the page passes or fails and provide evidence.
2. **Severity Ratings**: Rate each finding using a 4-level severity scale (cosmetic / minor / major / catastrophic). Justify each rating based on frequency, impact, and persistence.
3. **Evidence-Based Findings**: Each finding must reference a specific UI element or interaction pattern. Avoid vague observations; describe what the user sees, does, and expects.
4. **Recommendations**: For each finding, provide a specific, actionable recommendation. Group recommendations by effort level (quick fix / redesign / research needed).

## Output Contract
Produce a `usability-review.md` artifact with ## sections for the evaluation methodology, findings (organized by severity), and recommendations. Include a summary table of all findings with their severity and effort ratings.
