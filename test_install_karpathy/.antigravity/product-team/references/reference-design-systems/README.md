# Reference Design Systems

This library is inspiration-only. It is not the product's `project-ds-spec.md`.

Use it to pick strong starting references before building the product's own design system direction.

## How To Use It

- Read at most 3 company references per assignment.
- Borrow principles, not branding. Do not copy logos, brand names, product-specific illustrations, or exact named token systems.
- Translate what you borrow into the project's own `knowledge/project-ds-spec.md`.
- Prefer combining references that differ on one or two important axes instead of mashing together unrelated aesthetics.

## Quick Picks By Signal

| Signal | Strong References | Why |
|---|---|---|
| Monochrome minimal | `apple_design.md`, `vercel_design.md`, `cal_design.md` | Strong restraint, hierarchy, and whitespace discipline |
| Premium fintech trust | `stripe_design.md`, `revolut_design.md`, `wise_design.md`, `coinbase_design.md` | Reliable models for authority, polish, and data clarity |
| Developer-tool clarity | `vercel_design.md`, `supabase_design.md`, `hashicorp_design.md`, `mintlify_design.md` | Useful for technical products, structured content, and code-adjacent UI |
| Warm approachable productivity | `notion_design.md`, `airbnb_design.md`, `intercom_design.md`, `airtable_design.md` | Softer tones, accessible friendliness, and broad-market readability |
| Expressive playful B2B | `clay_design.md`, `framer_design.md`, `webflow_design.md`, `pinterest_design.md` | Higher personality, motion, and visual surprise without losing structure |
| Dark technical futurism | `x.ai_design.md`, `runwayml_design.md`, `elevenlabs_design.md`, `spacex_design.md` | Useful when the product needs high-contrast, cinematic, or frontier-tech tone |
| Enterprise structure | `ibm_design.md`, `mongodb_design.md`, `sentry_design.md`, `miro_design.md` | Strong references for consistency, predictable density, and scalable system behavior |

## Choose By Product Need

- Trust-heavy product: start with `stripe_design.md` or `wise_design.md`.
- Developer platform: start with `vercel_design.md`, `supabase_design.md`, or `mintlify_design.md`.
- Consumer or prosumer product: start with `airbnb_design.md`, `pinterest_design.md`, or `notion_design.md`.
- Bold brand expression: start with `clay_design.md`, `framer_design.md`, or `elevenlabs_design.md`.
- Dense enterprise workflow: start with `ibm_design.md`, `sentry_design.md`, or `mongodb_design.md`.

## Signal Heuristics

- Density:
  - Low: `apple_design.md`, `vercel_design.md`, `airbnb_design.md`
  - Medium: `notion_design.md`, `stripe_design.md`, `intercom_design.md`
  - High: `ibm_design.md`, `mintlify_design.md`, `sentry_design.md`
- Typography character:
  - Engineered: `vercel_design.md`, `stripe_design.md`, `hashicorp_design.md`
  - Warm editorial: `notion_design.md`, `airbnb_design.md`, `pinterest_design.md`
  - Experimental: `clay_design.md`, `runwayml_design.md`, `elevenlabs_design.md`
- Color energy:
  - Restrained: `apple_design.md`, `vercel_design.md`, `notion_design.md`
  - Controlled brand accent: `stripe_design.md`, `mintlify_design.md`, `intercom_design.md`
  - High energy: `clay_design.md`, `framer_design.md`, `webflow_design.md`
- Layout bias:
  - Hero-led single focus: `apple_design.md`, `spacex_design.md`
  - Structured marketing grid: `stripe_design.md`, `vercel_design.md`, `hashicorp_design.md`
  - Content-rich/productivity: `notion_design.md`, `mintlify_design.md`, `miro_design.md`
- Depth and radius bias:
  - Tight and precise: `stripe_design.md`, `vercel_design.md`, `ibm_design.md`
  - Soft and approachable: `airbnb_design.md`, `intercom_design.md`, `airtable_design.md`
  - Dramatic and expressive: `clay_design.md`, `runwayml_design.md`, `elevenlabs_design.md`

## Anti-Patterns

- Do not use this library as a substitute for the product's own system decisions.
- Do not build a direct clone of one company unless the assignment explicitly asks for mimicry.
- Do not let late-stage polish override the project's chosen `project-ds-spec.md`.
