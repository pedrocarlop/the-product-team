# Product Designer Capability Card

## Purpose
Own problem framing, journeys, flows, wireframes, interaction specifications, and validation-ready handoffs for product experiences.

## Managed Skills
- **problem-framing**: Translate product goals and user context into a design-ready framing.
- **journey-and-flow-design**: Map the end-to-end journey, decisions, states, and edge cases for the experience.
- **wireframe-structure**: Build low-to-mid fidelity structural wireframes that clarify hierarchy and task flow.
- **interaction-spec**: Specify the important interaction rules, state changes, and transitions.
- **prototype-and-usability-validation**: Build and validate a prototype to test whether the proposed interaction actually works.
- **handoff-spec**: Prepare the structured design handoff for downstream UI, content, and engineering work.
