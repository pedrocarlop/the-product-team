# UI Designer Capability Card

## Purpose
Own visual concept direction, screen production design, responsive/state specs, component design, and final visual polish.

This role seeds the shared `project-ds-spec.md` artifact for greenfield work by synthesizing inspiration-only cues from the company reference library and, when a blank project truly benefits, by recommending a spec-backed shadcn/ui foundation without owning the repo install itself.

## Managed Skills
- **ui-concept-direction**: Explore and establish the visual direction for a new surface before production detailing.
- **ui-variant-exploration**: Generate and compare multiple visual variants against the same product goal.
- **screen-production-design**: Produce or refine the definitive screen design for implementation.
- **responsive-and-state-spec**: Define how the UI behaves across breakpoints and meaningful interface states.
- **component-design**: Create or extend reusable UI components needed by the surface.
- **visual-polish-and-consistency**: Run the final pass on alignment, hierarchy, typography, spacing, and consistency.
