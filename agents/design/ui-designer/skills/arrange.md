# Arrange

Use this skill to fix layouts that feel crowded, flat, monotonous, or visually unbalanced.

## When to Use

- When hierarchy is weak because the composition does not guide the eye clearly
- When spacing, grouping, or alignment feels inconsistent
- When a surface needs stronger rhythm without inventing new components

## How to Use

Audit the screen as a composition problem first. Identify the primary focal point, secondary support content, and any elements competing for the same visual weight. Check grouping, grid behavior, edge alignment, whitespace usage, and the consistency of spacing tokens.

Strengthen the composition by clarifying grouping, tightening local clusters, opening intentional separation between sections, and using asymmetry or scale changes only where they improve hierarchy. If the layout still works only when viewed element-by-element, the arrangement is not doing enough.

## What to Produce

- Revised grouping and section structure
- Spacing and alignment rules that map cleanly to system tokens
- Any grid or container changes needed to improve rhythm and hierarchy
- A short explanation of the intended eye path through the surface

## Notes for UI Designer

Better arrangement should make the interface easier to read before any copy or styling changes are applied. Do not use extra containers or decoration to compensate for weak structure.
