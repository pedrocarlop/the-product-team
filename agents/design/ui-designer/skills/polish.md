# Polish

Use this skill to take a mostly solved surface through its final refinement pass before review or handoff.

## When to Use

- When the design is functionally correct but still feels slightly off
- When a feature needs finishing detail work before final delivery
- When multiple small inconsistencies are lowering the overall quality bar

## How to Use

Review the surface at full-page, section, and component levels. Check alignment, spacing rhythm, typography consistency, icon sizing, surface transitions, control states, form behavior, empty and error states, and responsive edge cases. Pay special attention to issues that are individually small but cumulative in effect.

Prioritize changes that improve perceived quality without reopening core design decisions. If the pass uncovers a deeper flow, copy, or systems problem, flag the owning role rather than quietly solving it as UI polish.

## What to Produce

- A concise punch list of exact UI corrections
- Any final state, breakpoint, or component refinements needed for handoff
- A short verification checklist covering visual detail, state coverage, and responsive behavior

## Notes for UI Designer

Polish is not decoration. It is the last pass that removes friction, ambiguity, and visual noise from an otherwise sound design.
