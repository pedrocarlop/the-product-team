# Colorize

Use this skill to improve a UI surface that feels too gray, flat, or emotionally neutral.

## When to Use

- When the interface lacks visual warmth, emphasis, or semantic color signaling
- When important actions or states are not separated clearly enough
- When the palette needs more range while staying within brand and system constraints

## How to Use

Map color to meaning first: action, emphasis, feedback, surface separation, data encoding, and ambient tone. Introduce color through semantic roles rather than isolated decoration. Keep neutrals, accents, and feedback colors working together as one system.

Check the result in all required themes and state combinations. Color additions must preserve contrast, hierarchy, and token discipline across default, hover, focus, active, disabled, loading, success, and error states.

## What to Produce

- A clear semantic color strategy for the surface
- The token or palette adjustments needed to support that strategy
- Specific guidance for accents, surfaces, text, borders, and any data visuals
- Any accessibility or theming constraints that must be preserved

## Notes for UI Designer

Color should communicate before it decorates. If a new color does not change understanding, hierarchy, or tone in a useful way, it probably does not belong.
