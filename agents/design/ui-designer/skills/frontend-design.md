# Frontend Design

Use this skill to establish the visual direction for a screen, feature, or UI surface before detailed component and state work begins.

## When to Use

- When a new surface needs a clear aesthetic direction before detailed UI specification
- When the current direction feels generic, inconsistent, or visually underdeveloped
- When a feature needs a stronger point of view across typography, color, spacing, hierarchy, and motion

## How to Use

Start from product context, not visual taste alone. Confirm the audience, core job-to-be-done, tone, and quality bar. Audit the existing design system, product patterns, and approved components before introducing new visual moves.

Choose a specific direction and make it concrete: hierarchy, type system, palette, surface treatment, spacing rhythm, and motion posture. Pressure-test the direction against common low-quality patterns such as generic card grids, decorative gradients, glass effects without purpose, over-centered layouts, weak focal points, and interchangeable font choices.

Define how the direction holds at real breakpoints and across real states. A strong art direction that collapses in loading, error, empty, or mobile views is not ready for handoff.

## What to Produce

- A concise visual direction statement for the surface
- The primary hierarchy decisions: focal points, typography, color emphasis, spacing rhythm, and motion posture
- The component and token choices that carry the direction consistently
- Any explicit anti-patterns to avoid while refining the UI spec

## Notes for UI Designer

Be bold, but stay accountable to the product's actual system and constraints. Distinctiveness should come from intentional design choices, not from novelty layered on top of weak hierarchy.
