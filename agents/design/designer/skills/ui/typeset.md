# Typeset

Use this skill to strengthen the typography system for a screen, flow, or component set.

## When to Use

- When the text hierarchy feels weak, inconsistent, or generic
- When readability suffers because of sizing, spacing, or density decisions
- When a surface needs more typographic craft without changing its overall structure

## How to Use

Audit the current type system in context: font choice, scale, weight distribution, line length, line height, label treatment, emphasis patterns, and responsive behavior. Confirm that headings, body, labels, captions, and data values each have clear jobs and do not overlap visually.

Refine typography as a system, not line-by-line styling. Improve contrast between levels, remove inconsistent exceptions, and check that text still works in dense layouts, long strings, and smaller breakpoints.

## What to Produce

- Updated type hierarchy rules for the surface
- Any font, weight, size, spacing, or emphasis adjustments required
- Guidance for truncation, wrapping, and responsive text behavior where needed

## Notes for UI Designer

Respect the existing design system first. Strong typography comes from clear hierarchy and disciplined repetition, not from arbitrary font changes.
