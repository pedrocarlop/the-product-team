# Quieter

Use this skill to calm a UI surface that feels loud, aggressive, or exhausting to look at.

## When to Use

- When too many elements compete for attention at once
- When contrast, color, or motion intensity overwhelms the task
- When a surface needs to feel more refined without becoming bland

## How to Use

Identify the elements currently demanding attention and decide which one truly deserves priority. Reduce competing accents, decorative contrast, dense borders, over-signaled surfaces, and unnecessary motion. Rebalance the page so the primary action stays clear while the rest of the interface becomes easier to scan and trust.

Quieting the surface should not flatten it. Preserve useful emphasis, semantic color, and enough contrast to keep the task legible and actionable.

## What to Produce

- A calmer attention model with fewer competing signals
- Specific reductions to color intensity, visual weight, and motion behavior
- Any revised spacing or grouping needed to support the quieter direction

## Notes for UI Designer

Refinement is not the same as removing personality. The goal is controlled emphasis, not visual anesthesia.
