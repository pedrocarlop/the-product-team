# Log

## Purpose

Keep `context.md` concise, current, and sufficient for future continuation without the full conversation.

## context.md should capture

- Project goal
- Current state
- Approved direction
- Active roles
- Key decisions and rationale
- Latest deliverables
- Open questions
- Next step

## TIMELINE.md

Update `logs/TIMELINE.md` when projects start, change status, or complete.
