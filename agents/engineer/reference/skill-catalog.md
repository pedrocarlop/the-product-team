# Reference Skill Catalog

Read this file first when you are staffed for orchestrated work.
It lists only the role-local skills in this folder and keeps descriptions short so you can scan cheaply.
Open only the matching skill files under `skills/`, then end your closing handoff with `Read <skill-paths> skills for this task.`

## Additional Skills

- `ground` — Ground decisions in the actual target repository and any named source-system or design-system repositories before proposing new behavior.
- `reuse` — Prefer existing approved patterns, helpers, and components over inventing new ones when solving a task.
- `trace` — Trace features, patterns, and decisions through the codebase so the real implementation path is understood end to end.
- `verify` — Confirm that the grounded facts, traced paths, and reused patterns are actually supported by the codebase before the decision is finalized.
