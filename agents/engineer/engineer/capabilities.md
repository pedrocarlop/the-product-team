# Engineer Capability Card

## Purpose
Own all implementation from UI components through backend services, mobile apps, and ML systems.

## Managed Skills
- **frontend**: translate, stateful, harden, tune
- **backend**: implement, model, observe, harden
- **fullstack**: model, ship, slice, wire
- **mobile**: adapt, bridge, compose, ship
- **ml**: evaluate, frame, monitor, serve
- **implementation**: apply-patch, figma-get-design-context, chrome-list-console-messages

## Input/Output Contract
- **Takes**: `logs/active/<project-slug>/context.md`, `logs/active/<project-slug>/deliverables/designer.md` (optional), `logs/active/<project-slug>/deliverables/product-lead.md` (optional)
- **Produces**: `logs/active/<project-slug>/plans/engineer.md`, `logs/active/<project-slug>/deliverables/engineer.md`

## External Tools (MCP)
- **figma**: Retrieve design context and screenshots.
- **chrome_devtools**: UI inspection, console monitoring, and snapshots.
- **github**: Repository management, code edits, and PR lifecycle.
- **linear**: Task status updates and issue tracking.
- **sentry**: Error tracking and performance monitoring.
- **vercel**: Deployment previews and build logs.
- **supabase**: Database management and edge functions.
