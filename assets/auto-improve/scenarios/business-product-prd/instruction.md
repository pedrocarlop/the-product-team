# Scenario: Business - PRD Clarity

## Task
Draft a PRD for an 'Automated Sprint Planner' AI feature within a task management tool.

## Requirements
1. **Problem Statement**: Clear definition of the 'Why'.
2. **Success Metrics**: At least 3 measurable KPIs.
3. **User Stories**: At least 5 detailed user stories with acceptance criteria.
4. **Technical Constraints**: Describe any AI-specific constraints (data privacy, model latency).
5. **Ambiguity Check**: The PRD must be actionable for a developer.

## Output Contract
Produce a `prd.md` document with these sections and high-density requirements.
